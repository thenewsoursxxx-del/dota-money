# DOTA MONEY — v1

Прогноз исхода матчей Tier-1 сцены Dota 2 и поиск **value против кэфа букмекера**.
Полностью статический сайт (расчёты в браузере) → бесплатно хостится на GitHub Pages и открывается как **Telegram Mini App**.
Сборка данных — на Node.js без внешних зависимостей.

## Идея

Приложение не просто показывает «кто фаворит» (это и так видно по кэфам). Оно считает **свою** вероятность исхода (Elo по недавним Tier-1 матчам) и сравнивает её с кэфом букмекера. Если модель даёт команде больше шансов, чем закладывает букмекер → это **value-ставка** (положительное матожидание). Именно так ловятся апсеты вроде Vici над Falcons или Yandex над Spirit.

## Структура

```
src/opendota.mjs        — клиент OpenDota API (Node): пагинация про-матчей, лиги, команды
src/build.mjs           — пайплайн (Node): Tier-1 матчи → Elo + тайминг → docs/data/dataset.json
src/build_draft.mjs     — драфт-данные (герои, контр-матчапы, пулы) → docs/data/draft.json
src/fetch_upcoming.mjs  — автозагрузка будущих Tier-1 матчей с Liquipedia → docs/data/upcoming.json
.github/workflows/      — GitHub Actions: автообновление матчей/рейтингов и драфта по расписанию
docs/                   — статический сайт (то, что деплоится на GitHub Pages)
  model.mjs        — общая модель: Elo, вероятность серии (BO1/BO3/BO5), value/EV/Kelly
  index.html
  styles.css
  app.js           — фронтенд, считает прогнозы client-side
  data/dataset.json  — собранные данные (создаётся build-data)
  data/upcoming.json — будущие матчи (редактируется вручную)
server.mjs         — локальный статический сервер для превью docs/
```

## Как запустить локально

```bash
# 1. Собрать/обновить данные с OpenDota (Tier-1 лиги + Elo + тайминг). ~30 сек.
npm run build-data

# 2. Собрать драфт-данные (ростеры, пулы игроков, контр-матчапы). ~5-6 мин.
npm run build-draft

# 3. Собрать мету v3 (синергии/контры/кривые силы из деталей матчей). ~15 мин.
npm run build-meta

# 4. Подтянуть будущие матчи с Liquipedia (Tier-1).
npm run fetch-upcoming

# 5. Локальное превью
npm start   # → http://localhost:5173
```

## Будущие матчи — автоматически (Liquipedia)

Матчи больше не нужно добавлять вручную:

```bash
npm run fetch-upcoming   # тянет предстоящие матчи с Liquipedia, оставляет только Tier-1
```

- **Источник**: публичный MediaWiki API Liquipedia (бесплатно, без ключа), один запрос.
- **Фильтр Tier-1**: обе команды должны быть в нашем датасете и проходить порог по Elo
  (сильнейшая ≥ `TIER1_TOP`, обе ≥ `TIER1_MIN`) — это отсекает квалы/Tier-2.
- **Имена**: несовпадения (напр. Liquipedia «BB Team» → наш «BoomBoys») разрешаются через
  алиасы в `src/fetch_upcoming.mjs`.
- **Кэфы**: `oddsA`/`oddsB` Liquipedia не даёт — их можно вписать вручную в `upcoming.json`,
  и они **сохраняются** при следующем автообновлении (матчинг по паре команд + дате).

### Автообновление (GitHub Actions)

- `.github/workflows/update-matches.yml` — каждые 6 часов: обновляет рейтинги/тайминг
  (`build-data`) и подтягивает матчи (`fetch-upcoming`), коммитит изменения в `docs/data`.
- `.github/workflows/update-draft.yml` — раз в неделю: пересобирает драфт-данные (`build-draft`).
- Оба можно запустить вручную через вкладку **Actions → Run workflow**.
- Требуется: Settings → Actions → General → Workflow permissions → **Read and write**.

## v2 — Драфт-интеллект (готово)

Добавлен слой анализа героев и драфта (`docs/data/draft.json`, вкладка «Драфт»):

- **Пулы героев игроков** и их предсказуемость (насколько узкий/читаемый пул).
- **Контр-матчапы героев** (pro-winrate герой vs герой) → «контрируемость» каждого героя.
- **Уязвимость матчапа с учётом соперника**: сигнатурный герой игрока + контра, которую соперник реально играет (`✓играет`) — это и есть логика «Collapse ↔ Undying».
- Небольшая **корректировка вероятности** прогноза на основе драфта.

Модель: `docs/draft.mjs`. Сборка: `src/build_draft.mjs`.

## v2.1 — Темп и тайминг (готово)

Для каждой команды по длительности Tier-1 игр считается стиль:

- **Средняя длина игры** и метка: `темповые` (≤37 мин) / `универсалы` / `лейтовые` (≥43 мин).
- **Винрейт по стадиям**: ранняя (<30 мин), средняя (30–40), лейт (>40) — видно, где команда сильна.
- **Темп**: закрывают быстро (победы короче поражений) или тянут в лейт.

Показывается строкой на карточке матча и подробным блоком «⏱ Стиль и темп» в калькуляторе.
Считается в `src/build.mjs` (`computeTiming`), отображается в `docs/app.js`.

## v3 — Live-драфт интеллект (готово)

Мощный анализатор матчапа по завершённому драфту. Вводишь 10 героев (+ опц. команды и
игроков) — движок оценивает по факторам, которые реально решают исход (подтверждено
исследованиями: draft-only ~55–65%, экономика @10мин → 80–90%+).

Данные: `src/build_meta.mjs` → `docs/data/meta.json`:
- **Мета героев** (`/heroStats`): pro pick/ban/win, статы, гейны.
- **Матрицы синергий и контр** из деталей ~900 Tier-1 матчей (`picks_bans`).
- **Кривые силы** (power curve): винрейт героя по длительности → ранняя/средняя/лейт.
- **Лайновая экономика**: средний нетворс героя к 10-й минуте.
- Курируемые теги: `docs/data/hero_knowledge.json` (тип урона, локдаун, BKB-pierce, архетип).

Движок: `docs/live_draft.mjs` (`scoreDraft` / `blend` / `applyEarlyGame` / `explain`).

Что показывает вкладка «Live-драфт»:
- Вероятность + калиброванная **уверенность** (по драфту потолок ~72%).
- **Кривые силы** обеих команд (ранняя/средняя/лейт) — «кто спайкует когда».
- **Разбор в стиле комментатора**: тайминги, контры с числами, пул-лок игрока
  («связаны руки»), баланс урона, синергии, стиль/архетип, лайновая экономика.
- **Стадия игры (~10 мин)**: вводишь нетворс/опыт/вышки/первую кровь — прогноз
  пересчитывается по экономике и уверенность растёт к 80–90%+.

Автообновление меты: `.github/workflows/update-meta.yml` (еженедельно).

### Авто-live (готово)

Кнопка **«Загрузить live-матч»** на вкладке Live-драфт тянет текущие игры напрямую из
OpenDota `/live` — без ключа и без прокси (эндпоинт отдаёт CORS). Автоматически:

- подставляет 10 героев по сторонам (Radiant → команда A, Dire → B);
- заполняет экономику: нетворс-лид (`radiant_lead`) и число снесённых вышек
  (декодит битмаску `building_state`);
- назначает игроков по `account_id` (если ростер команды есть в `draft.json`) →
  авто пул-лок в разборе;
- с включённым «авто-обновлением» раз в 20 c перезапрашивает состояние и
  пересчитывает вероятность вживую.

Приоритет — про-матчи (`league_id`); если про-игр в эфире нет, показываются топовые
паблики по MMR (для проверки движка 24/7). `/live` не отдаёт XP-лид и первую кровь —
эти поля остаются пустыми и в модели не участвуют.

### Value-беты в live

В live-драфте есть поля **Кэф на A / Кэф на B**. Если их заполнить, приложение
убирает маржу букмекера (vig), сравнивает честную вероятность со своей и показывает
рекомендацию с **EV** и долей банка по **Kelly**. Порог value — EV ≥ 5%.

### Бэктест (`npm run backtest`)

Честный walk-forward по истории Tier-1: для каждого матча прогноз строится
**только по прошлым играм**, затем Elo обновляется (out-of-sample). Метрики пишутся
в `docs/data/backtest.json`. На ~1100 оценённых матчах:

- **Accuracy** (угадан фаворит): ~61%
- **Brier** 0.234 против базлайна 0.249 → **Brier Skill Score +6%** (модель лучше «просто средней»)
- Калибровка адекватная: где модель говорит 60% — реально ~64–69%, где 70% — ~77%.
- Radiant win rate 53.6% (сторона реально имеет перевес).

Что это значит честно: **предматчевая модель (Elo) — умеренно лучше монетки, а не оракул на 90%.**
Высокая точность (80–98%) появляется только когда подгружается экономика live-игры —
т.е. это уже «что происходит сейчас», а не «кто победит до начала».

Ограничение: у нас **нет исторических кэфов**, поэтому ROI против рынка бэктестом
пока не доказан — метрики измеряют точность прогноза, не прибыль ставок.

### Обучаемая ML-модель (`npm run train`)

Вместо ручных весов исход матча предсказывает **калиброванная логистическая регрессия**,
обученная на истории Tier-1 (`train.json` из `build-data` → `model.json`). Фичи —
только A-минус-B по состоянию **до игры**:

- `eloDiff` — **time-decay Elo** (регрессирует к среднему при простое → рейтинг «двухлетней давности» не тянет);
- `form10Diff`, `form45Diff` — форма за 10 игр / 45 дней (это игры **текущего** ростера в **текущей** мете → смена состава и сдвиг меты учитываются автоматически);
- `actDiff` — активность за 45 дней;
- `rustDiff` — простой (дни без игр).

Обучение **взвешивает свежие матчи экспоненциально сильнее** (полураспад 90 дней),
поэтому веса отражают актуальную мету. Сторона (radiant edge ~53.6%) уходит в intercept.

Честная оценка (холдаут на свежих ~220 матчах, out-of-sample):

| Модель | Accuracy | Brier | LogLoss |
|---|---|---|---|
| Elo baseline | 68.2% | 0.2151 | 0.6209 |
| **ML (калиброванная)** | **70.0%** | **0.2105** | **0.6100** |

ML‑модель немного точнее и лучше калибрована, чем чистый Elo, и — главное — **устойчива
к устаревшим рейтингам** за счёт decay + формы. На walk-forward (5 фолдов) ML ≈ Elo:
на таком объёме (~1100 матчей) выигрыш скромный, но реальный. Клиент сервит модель через
`docs/model_ml.mjs` (те же фичи офлайн и онлайн — без расхождений).

Переобучение автоматическое: `update-matches.yml` после `build-data` вызывает `train`.

### Журнал ставок и ROI (`npm run roi`)

Раз исторических кэфов нет, копим свои **going-forward**. В результате Live/Калькулятора
есть кнопка **«Записать ставку»** — фиксирует момент: команды, формат, вероятность модели,
введённый кэф и выбранную сторону (по EV). Хранится локально (localStorage), экспорт/импорт
JSON — в блоке **📒 Журнал ставок** (внизу Live).

Сверка по факту: экспортируй журнал (`dota-money-bets.json`), положи в корень и запусти
`npm run roi`. Скрипт по `match_id` (есть у ставок из Live) тянет результат из OpenDota,
проставляет win/loss и считает **winrate + ROI** (флэт 1 ед.). Так через десяток-другой
матчей появится честная цифра, бьём ли мы рынок. Ставки из Калькулятора без `match_id`
результат не получают автоматически — их итог можно проставить руками.

## Вкладки

- **Матчи** — будущие Tier-1 игры с value-бейджем.
- **Калькулятор** — быстрый прогноз «команда + кэф» до стадии пиков.
- **Live-драфт** — автозагрузка live-матча / ручной ввод 10 героев → полный разбор + журнал.
- **Рейтинг** — Elo команд.
