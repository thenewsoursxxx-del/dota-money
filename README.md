# DOTA MONEY — v1

Прогноз исхода матчей Tier-1 сцены Dota 2 и поиск **value против кэфа букмекера**.
Полностью статический сайт (расчёты в браузере) → бесплатно хостится на GitHub Pages и открывается как **Telegram Mini App**.
Сборка данных — на Node.js без внешних зависимостей.

## Идея

Приложение не просто показывает «кто фаворит» (это и так видно по кэфам). Оно считает **свою** вероятность исхода (Elo по недавним Tier-1 матчам) и сравнивает её с кэфом букмекера. Если модель даёт команде больше шансов, чем закладывает букмекер → это **value-ставка** (положительное матожидание). Именно так ловятся апсеты вроде Vici над Falcons или Yandex над Spirit.

## Структура

```
src/opendota.mjs        — клиент OpenDota API (Node): пагинация про-матчей, лиги, команды
src/build.mjs           — пайплайн (Node): Tier-1 матчи → Elo + тайминг → docs/data/dataset.json
src/build_draft.mjs     — драфт-данные (герои, контр-матчапы, пулы) → docs/data/draft.json
src/fetch_upcoming.mjs  — автозагрузка будущих Tier-1 матчей с Liquipedia → docs/data/upcoming.json
.github/workflows/      — GitHub Actions: автообновление матчей/рейтингов и драфта по расписанию
docs/                   — статический сайт (то, что деплоится на GitHub Pages)
  model.mjs        — общая модель: Elo, вероятность серии (BO1/BO3/BO5), value/EV/Kelly
  index.html
  styles.css
  app.js           — фронтенд, считает прогнозы client-side
  data/dataset.json  — собранные данные (создаётся build-data)
  data/upcoming.json — будущие матчи (редактируется вручную)
server.mjs         — локальный статический сервер для превью docs/
```

## Как запустить локально

```bash
# 1. Собрать/обновить данные с OpenDota (Tier-1 лиги + Elo). ~30 сек.
npm run build-data

# 2. Собрать драфт-данные (герои, контр-матчапы, ростеры, пулы игроков). ~5-6 мин.
npm run build-draft

# 3. Локальное превью
npm start   # → http://localhost:5173
```

## Будущие матчи — автоматически (Liquipedia)

Матчи больше не нужно добавлять вручную:

```bash
npm run fetch-upcoming   # тянет предстоящие матчи с Liquipedia, оставляет только Tier-1
```

- **Источник**: публичный MediaWiki API Liquipedia (бесплатно, без ключа), один запрос.
- **Фильтр Tier-1**: обе команды должны быть в нашем датасете и проходить порог по Elo
  (сильнейшая ≥ `TIER1_TOP`, обе ≥ `TIER1_MIN`) — это отсекает квалы/Tier-2.
- **Имена**: несовпадения (напр. Liquipedia «BB Team» → наш «BoomBoys») разрешаются через
  алиасы в `src/fetch_upcoming.mjs`.
- **Кэфы**: `oddsA`/`oddsB` Liquipedia не даёт — их можно вписать вручную в `upcoming.json`,
  и они **сохраняются** при следующем автообновлении (матчинг по паре команд + дате).

### Автообновление (GitHub Actions)

- `.github/workflows/update-matches.yml` — каждые 6 часов: обновляет рейтинги/тайминг
  (`build-data`) и подтягивает матчи (`fetch-upcoming`), коммитит изменения в `docs/data`.
- `.github/workflows/update-draft.yml` — раз в неделю: пересобирает драфт-данные (`build-draft`).
- Оба можно запустить вручную через вкладку **Actions → Run workflow**.
- Требуется: Settings → Actions → General → Workflow permissions → **Read and write**.

## v2 — Драфт-интеллект (готово)

Добавлен слой анализа героев и драфта (`docs/data/draft.json`, вкладка «Драфт»):

- **Пулы героев игроков** и их предсказуемость (насколько узкий/читаемый пул).
- **Контр-матчапы героев** (pro-winrate герой vs герой) → «контрируемость» каждого героя.
- **Уязвимость матчапа с учётом соперника**: сигнатурный герой игрока + контра, которую соперник реально играет (`✓играет`) — это и есть логика «Collapse ↔ Undying».
- Небольшая **корректировка вероятности** прогноза на основе драфта.

Модель: `docs/draft.mjs`. Сборка: `src/build_draft.mjs`.

## v2.1 — Темп и тайминг (готово)

Для каждой команды по длительности Tier-1 игр считается стиль:

- **Средняя длина игры** и метка: `темповые` (≤37 мин) / `универсалы` / `лейтовые` (≥43 мин).
- **Винрейт по стадиям**: ранняя (<30 мин), средняя (30–40), лейт (>40) — видно, где команда сильна.
- **Темп**: закрывают быстро (победы короче поражений) или тянут в лейт.

Показывается строкой на карточке матча и подробным блоком «⏱ Стиль и темп» в калькуляторе.
Считается в `src/build.mjs` (`computeTiming`), отображается в `docs/app.js`.
